Algebra 2 Excellence Bible — Part 6: Exponents & Logs (Refined)

Core principle: A logarithm is an exponent written sideways — understand the inverse and the constraints.

Logs undo exponentials. Master the undoing, and you master growth & decay.

F16 — Exponent Rules (Grammar of Growth)

Spiral link: PEMDAS & repeated multiplication. Exponents are shorthand for multiplying the same factor.

Tier 1 — MADSPM (Concise rules)

  • Multiply bases: add exponents: x²·x³=x⁵
  • Divide bases: subtract exponents: x⁵/x²=x³
  • Power to a power: multiply exponents: (x²)³=x⁶

Always check bases are the same before applying these rules.

Tier 2 — Negative & Fractional Exponents

Negative exponents: flip the base: x⁻² = 1 / x².

Fractional exponents: denominator = root, numerator = power.

x^(m/n) = (x^(1/n))^m = (ⁿ√x)^m = ⁿ√(x^m)

Important: If n is even, the real root requires x ≥ 0. For negative x and even roots, the result is complex.

F17 — Definition & Basic Constraints

Spiral link: Inverse functions — logs undo exponentials; domain & base matter.

Tier 1 — The Hook (Definition)

Definition: y = log_b(x) ⇔ b^y = x.

Domain & base rules: argument x must be positive (x>0). Base b must satisfy b>0 and b≠1.

Key caution: log rules (product/quotient/power) apply only when you are manipulating logs of the same base. You cannot do algebra across different bases without converting.

Tier 2 — Common & Natural Logs

  • log(x) usually means base 10 in many contexts (common log).
  • ln(x) means base e (natural log), where e ≈ 2.7182818. Used for continuous growth/decay and calculus.

F18 — Log Properties (Expand & Condense)

Spiral link: Exponent rules — logs behave like exponents because they are exponents.

Tier 1 — The Three Core Laws

Name Property Why (exponent view)
Product log_b(AB)=log_b(A)+log_b(B) Because b^{u+v}=b^u·b^v
Quotient log_b(A/B)=log_b(A)-log_b(B) Because b^{u-v}=b^u/b^v
Power log_b(A^k)=k·log_b(A) Bring exponent down

Common trap: log(a + b) ≠ log a + log b. The properties only work for multiplication/division/powers.

Tier 3 — Change of Base

If you only have natural or common logs available:

log_b(x) = ln(x) / ln(b) = log(x) / log(b)

Use this to convert between bases before applying rules.

Practice, Pitfalls & Worked Examples

Tier 1 — Quick Practice

  • Expand: log(x^2 y / z) → 2 log x + log y − log z.
  • Condense: ln 5 + ln 2 → ln 10.
  • Solve: e^(2x)=7 → x = (1/2) ln 7.

Tier 3 — Common Pitfalls

  • Applying log rules across different bases without converting.
  • Attempting to take even root of negative number (real vs complex).
  • Forgetting base constraints: base cannot be 1 or negative, argument must be positive.
  • Assuming log(a+b)=log a + log b — false.

Worked Example — Solve

Example: Solve 5^(x − 2) = 25.

  1. Recognize 25 = 5².
  2. So 5^(x − 2) = 5² → equate exponents: x − 2 = 2.
  3. Therefore x = 4.

Example (log eq): Solve log_3(x) = 4.

  1. Rewrite: 3^4 = x.
  2. x = 81.
